1) Unidades CSS

Las unidades definen cómo se calculan tamaños y espacios. Es clave elegir bien según el contexto (fijo, relativo al root, al padre, al contenedor o al viewport).

px (absoluto)

100px

Medida fija. Útil para bordes o íconos exactos, pero no es flexible.

rem (root em)

rem (root em)

6rem

Relativo al font-size del <html>. Escala todo el sitio si se cambia ese valor.

em (padre)

6em

Relativo al tamaño de fuente del padre. Puede acumularse al anidar.

% (contenedor)

60%

Se calcula respecto al tamaño del contenedor inmediato.

vw/vh (viewport)

20vh
20dvh

En mobile: dvh y svh corrigen el “salto” de altura cuando aparecen/ocultan las barras del navegador.

  • rem para tipografía y espacios globales.
  • em para tamaños internos de componentes.
  • % para layouts fluidos dentro de un padre.
  • dvh en pantallas completas para evitar saltos en mobile.

3) Safe-area

En dispositivos con notch o gestos, los elementos fijos pueden quedar tapados. env(safe-area-inset-*) agrega espacio dinámico para evitarlo.

Botón fijo Reservar

Se usa en headers, footers y barras flotantes para garantizar usabilidad en iOS/Android modernos.
